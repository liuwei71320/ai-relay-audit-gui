# API Relay Security Audit Tool 升级日志 (v3.5)

本项目是基于原始开源项目 [toby-bridges/api-relay-audit](https://github.com/toby-bridges/api-relay-audit/tree/master) 进行的深度二次开发版本。在原有审计逻辑的基础上，我们进行了如下重构与功能增强：

## 1. 全新 Web 控制台 (Streamlit 架构)
*   **交互式界面**：弃用了单一的命令行模式，引入了可视化 Web 界面。
*   **实时监控**：审计过程中可实时查看进度条、当前执行步骤及即时反馈。
*   **持久化显示**：通过 `session_state` 技术解决了测试结束后报告闪烁消失的 UI Bug，确保报告与下载按钮稳定可见。
*   **防误触机制**：测试期间会自动禁用启动按钮，并在侧边栏提供详细的风险维度说明。

## 2. 经济性优化 (Token 节流)
针对用户反馈的 Token 消耗问题，引入了多项“脱水”优化：
*   **⚡ 快速上下文测试**：新增“快速模式”（UI 默认开启），将 Step 7 的扫描阶梯从最高 800K 缩减至 200K 字符。该模式能覆盖 99% 的常用模型边界，同时**节省约 70% 的输入 Token**。
*   **极简指令响应**：优化了 Step 14 的探测 Prompt，强制模型使用“简短的一句话”回答，并将 `max_tokens` 从 200 下调至 **50**，大幅削减了不必要的推理 Token 支出。

## 3. Windows 环境性能与稳定性修复
*   **通信协议重构**：将 `curl` 数据传输改为 `stdin` (标准输入) 模式，彻底绕过 Windows 32KB 命令行限制，确保超长测试不崩溃。
*   **探测算法优化**：修复了格式自动探测在异常情况下的“双重超时死循环”。设置了 30s 快速失败机制与状态锁定，将异常测试耗时从 30 分钟压缩至秒级。
*   **CURL 逻辑精整**：修正了头部与数据传输冲突，确保 100% 的 API 兼容性。

## 4. 深度审计模块：Step 14 模型真实性与纯度测试
为了应对中转站日益隐蔽的欺诈手段，新增了专用的“纯度测试”模块：
*   **版本欺诈识别 (Spoofing)**：利用 2024 年以后的“知识断层指纹”检测，识破用旧模型冒充新模型（如 Haiku 冒充 Sonnet）的行为。
*   **智力退化检测 (Degraded)**：内置了一组专门设计的“反直觉逻辑陷阱”，通过压力测试识别模型是否因过度量化（Q2/Q4）或廉价路由导致的逻辑能力下降。

## 5. 易用性改进
*   **一键启动脚本 (`start_gui.bat`)**：支持全中文环境显示，自动检查并配置 Python 环境及依赖库。
*   **本地报告管理**：生成的 `report.md` 支持一键下载至本地，文件名自动包含时间戳。
